Проект состоит из:
WebApi - backend на ASP.NET Core 8, который отвечает за создание и хранение устройств и их сессий, получение данных, а также формирование JSON-бэкапа.
SPA - приложение для просмотра списка устройств, их сессий и скачивания JSON-бэкапа.
senderSPA — вспомогательное приложение для отправки новых сессий для выбранного устройства.

База данных — PostgreSQL, схема создаётся автоматически при запуске WebAPI.

Варианты запуска:
1. Запуск через Docker

Необходимо в корне репозитория выполнить
"docker compose up"

После успешного запуска будут доступны:
- PostgreSQL: "localhost:5432" (БД "device_db", пользователь "postgres", пароль "2025");
- Backend WebApi: "http://localhost:5273";
- Swagger UI для WebApi: "http://localhost:5273/swagger";
- Основное SPA: "http://localhost:4200";
- senderSPA: "http://localhost:4300"."

2. Локальный запуск
2.1. Нужно первоначально настроить БД (параметры заданы в конфиге)
2.2. Затем запустить WebAPI и SPA

По умолчанию API будет доступен на "http://localhost:5273" (порт можно изменить в "launchSettings.json"). 
Swagger UI для тестирования и документации API: "http://localhost:5273/swagger".

Приложение SPA будет доступно по адресу "http://localhost:4200" (запросы к API проксируются через "proxy.conf.json").

Приложение senderSPA будет доступно по адресу "http://localhost:4300" (оно также использует proxy-конфигурацию для обращения к API).

WebApi:
- создаёт и обновляет устройства (добавление доступно только на backend);
- создаёт сессии устройств;
- дает список устройств и сессий по устройству;
- формирует JSON-бэкап устройств и сессий.

SPA:
- показывает список устройств;
- позволяет открыть страницу устройства и посмотреть его сессии;
- даёт возможность скачать JSON-бэкап.

senderSPA:
- загружает список устройств;
- позволяет выбрать устройство, указать интервал работы и версию;
- отправляет данные о новой сессии в WebApi.
